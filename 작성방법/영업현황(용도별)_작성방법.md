## 📋 이 표는 이렇게 만들어집니다

**⚠️ 먼저 알아두실 점**: 이 화면은 **"상품"이 아니라 "용도" 기준**으로 집계합니다.
"상품별" 화면(가정용/일반용/업무용/산업용/열병합용)과는 분류 기준 자체가 다른 별도의 표입니다.

- **상품**: 계약 시 적용되는 요금제 종류 (예: 취사난방용, 개별난방용 등)
- **용도**: 계량기를 등록할 때 **업종을 선택하면 그에 따라 자동으로 결정되는 값**으로,
  업종분류와 함께 함께 정해집니다. 즉 영업담당자가 "업종"을 고르면
  시스템이 "업종분류"와 "용도"를 같이 채워주는 구조입니다.

같은 계량기라도 "상품"과 "용도"는 서로 다른 컬럼에서 따로 관리되는 값이며,
이 화면은 그중 **용도** 컬럼만 사용합니다.

> 이 표에서 용도가 가장 중요하게 쓰이는 부분은 **공동주택과 단독주택을 구분하는 기준**이라는 점입니다.
> (자세한 보정 로직은 2번 항목 참고)

---

### 1. 용도는 어떻게 구성되나요?

세부 그룹핑 없이, CSV의 **용도 컬럼 값을 그대로** 사용합니다.

| 표시 컬럼 | 비고 |
|---|---|
| 공동주택 | |
| 단독주택 | 업종이 `다세대` 또는 `연립`을 포함하면 용도를 강제로 "단독주택"으로 보정 |
| 영업용 | |
| 업무용 | |
| 산업용 | |
| 열병합용 | |

표에는 위 6개 컬럼 외에 **소계**(공동주택+단독주택)와 **총합계**(6개 항목 전체 합)가
추가로 표시됩니다. (최종 컬럼 순서: 공동주택 · 단독주택 · 소계 · 영업용 · 업무용 · 산업용 · 열병합용 · 총합계)

> "상품별" 화면처럼 매핑 안 된 상품이 별도 컬럼으로 늘어나는 구조가 아니라,
> 위 6개 용도로 **고정된 컬럼**만 사용합니다. 6개 용도에 해당하지 않는 값은 집계에 잡히지 않습니다.

---

### 2. 단독주택 보정 로직 (★ 가장 중요한 부분)

용도 컬럼은 기본적으로 업종 선택에 따라 자동으로 정해지지만,
**공동주택과 단독주택을 구분하는 기준** 하나는 별도로 한 번 더 보정합니다.

업종 컬럼(`업종` 또는 `업종분류`)에 **"다세대" 또는 "연립"**이라는 글자가 포함된 건은
원래 용도 값이 무엇이든 상관없이 **"단독주택"으로 강제 변경**한 뒤 집계합니다.
(다세대/연립주택은 통계상 단독주택과 같은 그룹으로 보기 위한 보정입니다)

---

### 3. 제외 항목

"상품별" 화면과 달리, **별도로 제외하는 상품/용도명은 없습니다.**
단, 공급전(공급 건수) 집계에서만 아래 예외가 적용됩니다.

| 적용 대상 | 내용 |
|---|---|
| 공급전 | 업종이 `CES_죽곡`인 건은 건수를 0으로 처리 (공급전 실적에서만 제외) |
| 시설전 | 별도 제외 없음 (전체 포함) |

---

### 4. 각 항목의 의미

| 항목 | 의미 |
|---|---|
| 계약전 | 공급계약전 건수에서 폐전 건수를 뺀 순증감 |
| 시설전 | 공급계약이 끝나고 공사를 마친 뒤, 안전시설검사를 완료한 계량기 |
| 공급전 | 최초 사용계약이 완료된 계량기 |
| 폐전 | 폐전 건수 |
| 용도변경 | 용도 간 이동 (이전 용도 → 변경 용도) |

각 항목은 **당월 / 전월 / 증감** 3줄로 비교되고, 용도변경은
**유입(다른 용도에서 들어옴) / 유출(다른 용도로 빠짐) / 합계**로 표시됩니다.

---

### 5. 핵심 지표 — 실적 / 계획 / 달성률

> **실적 = 공급전 증감 + 폐전 증감 − 용도변경 합계** (열병합용 포함)

추가로 이 화면에는 "상품별" 화면에는 없는 **계획 대비 비교**가 들어 있습니다.

- **계획**: 별도로 관리되는 구글시트(연월별 목표 건수)에서 해당 연월 행을 가져온 값
- **계획 대비**: 실적 − 계획
- **달성률**: 실적 ÷ 계획 (%로 표시)

계획 시트에 해당 연월 데이터가 없으면 계획/달성률/계획 대비 행은 표에서 자동으로 생략됩니다.
